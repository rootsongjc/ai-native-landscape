# Deployment

## Overview

The site deploys to Cloudflare Pages as a fully static site. Every page is pre-rendered at build time by Astro. There is no server-side runtime.

- **Hosting**: Cloudflare Pages
- **Domain**: landscape.jimmysong.io
- **Build output**: `dist/` (generated by `astro build`)
- **Node.js**: 20.x

## Local Development

```bash
# Install dependencies
npm install

# Start dev server
npm run dev
# Opens at http://127.0.0.1:4173

# Preview production build
npm run build
npm run preview
# Previews at http://127.0.0.1:4173
```

## Build Pipeline

The `npm run build` command executes the full pipeline in sequence:

```bash
npm run validate && npm run build:index && npm run export && astro build
```

| Step | Script | Output |
|---|---|---|
| Validate | `scripts/validate-data.mjs` | Validates all project files against schema and taxonomy |
| Build index | `scripts/build-index.mjs` | `public/data/index.json` |
| Export data | `scripts/export-data.mjs` | `public/exports/projects.json`, `public/exports/projects.csv` |
| Astro build | `astro build` | `dist/` (static HTML, CSS, JS) |

If any step fails, the build stops and the site is not generated.

## CI/CD

### GitHub Actions

`.github/workflows/ci.yml` runs on every push to `main` and on every pull request:

```yaml
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run build
```

This runs the full build pipeline (validate, index, export, Astro build). A failing validation or build blocks the PR from merging.

### Cloudflare Pages

Cloudflare Pages is configured to:

1. **Trigger**: Build on every push to `main`.
2. **Build command**: `npm run build`
3. **Output directory**: `dist`
4. **Node.js version**: 20

The deployment is automatic after a merge to `main`. No manual deployment step is required.

#### Cloudflare Pages Configuration

| Setting | Value |
|---|---|
| Framework preset | Astro |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Node.js version | 20 (set via environment variable `NODE_VERSION=20`) |
| Root directory | `/` (default, since the repo is the project root) |

## Environment Setup

### Requirements

- Node.js 20+
- npm 9+

### No Environment Variables Required

The build is fully self-contained. Project data comes from local markdown files in `data/projects/`. Scores are fetched at runtime by the client from the external score API (`ai-oss-rank-worker.jimmysong.io`), so no API keys are needed during build.

## Data Pipeline Scripts

These scripts are used for data management, not deployment:

| Script | Command | Purpose |
|---|---|---|
| Import | `npm run import:website` | Import project data from the main jimmysong.io website |
| Migrate to markdown | `npm run migrate:projects-markdown` | Convert imported data into bilingual markdown files |
| Migrate taxonomy | `npm run migrate:taxonomy` | Update category taxonomy structure |
| Validate | `npm run validate` | Validate all projects against schema and taxonomy |
| Build index | `npm run build:index` | Generate the search index JSON |
| Export | `npm run export` | Export project data as JSON and CSV |

### Import Workflow (for maintainers)

When importing a batch of new projects:

```bash
# 1. Import raw data from the main site
npm run import:website

# 2. Convert to bilingual markdown format
npm run migrate:projects-markdown

# 3. Update taxonomy if new categories are needed
npm run migrate:taxonomy

# 4. Validate everything
npm run validate

# 5. Build and test locally
npm run build
npm run preview
```

## Rollback

Cloudflare Pages retains deployment history. To roll back:

1. Go to the Cloudflare Pages dashboard for the project.
2. Select a previous successful deployment.
3. Click "Rollback to this deployment".

Alternatively, revert the commit on `main` and push -- Cloudflare Pages will automatically redeploy.
