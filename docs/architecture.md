# Architecture

## Overview

AI Native Landscape is a static site built with Astro 6 that catalogs 600+ open source AI projects. It generates a bilingual (Chinese/English) landscape with scoring, search, filtering, and category browsing. The site is fully pre-rendered at build time with no client-side framework.

- **Framework**: Astro 6 (static output mode)
- **Language**: ES modules throughout (`"type": "module"`)
- **Data format**: Markdown files with YAML frontmatter
- **Validation**: JSON Schema (ajv)
- **Deployment**: Cloudflare Pages
- **Interactivity**: Vanilla JS in `<script>` tags inside Astro components

## Data Layer

### Project Files

Each project is stored as a pair of markdown files in `data/projects/`:

- `<slug>.en.md` -- English entry
- `<slug>.zh.md` -- Chinese entry

The files share the same YAML frontmatter. At build time, `loadProjectDocuments()` merges both files into a single project object, using `description.zh` from the Chinese file and `description.en` from the English file.

Required frontmatter fields:

| Field | Type | Notes |
|---|---|---|
| `name` | string | Project display name |
| `slug` | string | URL-safe identifier (`^[a-z0-9-]+$`) |
| `repo` | URI | GitHub repository URL (required) |
| `category` | string | Must match a key in `categories.yaml` |
| `description` | object | Must contain both `zh` and `en` strings |

Optional fields: `homepage`, `license`, `subCategory`, `tags`, `author`, `ossDate`, `featured`, `thumbnail`, `source`, `status`, `score`.

The `score` object contains: `health`, `activity`, `community`, `popularity`, `quality`, `sustainability` -- all 0-100.

### Schema Validation

`data/schema.project.json` defines the JSON Schema. `scripts/validate-data.mjs` uses ajv with format validators. Validation checks:

1. Schema compliance (required fields, types, patterns)
2. Category and subCategory keys exist in taxonomy
3. subCategory belongs to its declared parent category
4. Both `description.zh` and `description.en` are present
5. `repo` is a valid GitHub URL
6. No duplicate slugs or repository URLs
7. Warns on category concentration (>35%) and empty subcategories

### Taxonomy

`data/categories.yaml` defines 8 top-level categories, each with 4-8 subcategories:

| Category | Subcategories |
|---|---|
| Agent Systems | Agent Frameworks, Orchestration & Workflows |
| Knowledge & Context | Memory & Context, Vector Databases, Retrieval & Indexing, Document Processing, Knowledge Graphs, Data Connectors |
| Models & Modalities | Foundation Models, Model Toolkits, Multimodal, Audio & Speech, Image & Video Generation |
| Inference & Runtime | Model Serving, Inference Runtimes, Sandboxes & Execution, GPU Acceleration, Edge & Local, LLM Routing & Gateways |
| Training, Evaluation & Optimization | Training Frameworks, Finetuning & Alignment, Experiment & MLOps, Evaluation & Benchmarks, Observability & Monitoring, Prompt Quality, Safety & Guardrails, Optimization & Simulation |
| Developer Tooling | MCP & Tool Protocols, Coding Agents, IDE & CLI Tools, SDK Frameworks, Browser Automation, Developer Utilities |
| Applications & Experience | Chat Interfaces, Desktop Clients, Workflow Automation, Low-code Builders, Productivity Tools |
| Platform & Infrastructure | Cloud Native AI, Data Platforms, Security & Policy, Deployment & Operations |

The file also contains `legacyAliases` that map old category keys to their current equivalents for backward-compatible URL routing.

### Auto-Classification

`scripts/lib.mjs` contains two rule sets for automatically resolving categories and subcategories from project tags:

- **`TAG_PRIORITY_RULES`** -- Maps tags to subcategories. Rules run in order; first match wins.
- **`SUBCATEGORY_RULES`** -- Maps descriptive terms in project metadata to subcategories.

Projects with `category` and `subCategory` already set use those values directly. Otherwise, `resolveProjectTaxonomy()` applies the rules.

## Scoring System

Scores are fetched at runtime from an external API at `https://ai-oss-rank-worker.jimmysong.io`. The API returns per-project scores.

Four scoring dimensions (0-100 each):

| Dimension | Description |
|---|---|
| Activity | Commit frequency, issue response time, release cadence |
| Community | Contributors, PR merge speed, discussion volume, ecosystem plugins |
| Impact | Stars, forks, dependency graph, external references |
| Sustainability | Maintenance history, team stability, commercial backing, roadmap |

Overall health is a weighted sum of these dimensions. Tier labels:

| Score Range | Label |
|---|---|
| 90-100 | Must Watch |
| 80-89 | Strong Recommendation |
| 70-79 | Worth Trying |
| 60-69 | Monitor Progress |
| 0-59 | Explore Carefully |

## Internationalization

Bilingual content is managed through paired files. Language routing uses `/en/` and `/zh/` path prefixes. The `UI` object in `src/lib/landscape.mjs` contains all UI strings for both languages.

Helper functions:

- `localizedPath(path, lang)` -- Prepends language prefix
- `alternatePath(path, lang)` -- Returns the URL for the other language
- `ui(lang)` -- Returns the full UI string map for a language

## Components

| `LandscapeGrid.astro` | Main page orchestrator: combines toolbar, category sections, and cards |
| `LandscapeToolbar.astro` | Search input, sort dropdown, status filter, view toggle, export button |
| `LandscapeCategory.astro` | Renders a category section with its subcategories and project cards |
| `LandscapeCard.astro` | Single project card: icon, name, score badge, tags, status |
| `LandscapeHeatmap.astro` | Alternative view showing project distribution by category x score tier |
| `LandscapeIndex.astro` | Full-page search and browse interface |
| `ProjectDetail.astro` | Detailed project page: scores breakdown, GitHub metrics, description, links |
